# Example CS Context

## Service Overview
- Replace this file with your own support knowledge base.
- Keep product summary, FAQ, policies, and known issues in Markdown.
- The dashboard scans this directory and makes the context available to the CS Helper tab.

## FAQ
### Q: Where should I start?
A: Begin with a short service description and 5-10 high-frequency customer questions.

### Q: Can I keep multiple products here?
A: Yes. Create one Markdown file per product or team.

## Response Policy
- Use a clear and consistent support tone.
- Add escalation rules for billing, bugs, and account issues.
- Note any legal or refund constraints explicitly.

## Known Issues
- Add active incidents or recurring edge cases here.
